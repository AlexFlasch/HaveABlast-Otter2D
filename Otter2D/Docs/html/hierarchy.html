Otter.ButtonComponent used for interpreting input as a button. It can recieve input from multiple sources including keyboard, mouse buttons, or joystick buttons and axes. The button input can also be controlled in code.
Otter.ControllerComponent representing a group of Button and Axis classes. The controller updates all buttons and axes manually. This is used by the Session class to manage player input
Otter.CounterComponent used for a value with built in min, max, and wrapping capabilities. Can be useful for making menus.
Otter.GridCounterCounter in which the value can be moved in both an X and Y direction. Probably most useful for making menus that are grids which the player can move around in.
Otter.IRCComponent for connecting to and interacting with an IRC server. Contains only very basic functionality. Probably not very useable yet!
Otter.LerperComponent that will slowly interpolate a value toward a target using a speed and acceleration. This component can move the value and does not know about time at all.
Otter.MovementA base class for Movement Components to extend.
Otter.BasicMovementMovement Component that can be used by an Entity to provide simple top-down style movement. This class requires an Axis component be assigned to it, and that Axis must be also updated by another source. A simple way to do this is to just have the Entity have an Axis component as well as the movement component, and then pass a reference to the axis into the movement
Otter.PlatformingMovementMovement Component that adds platforming movement behavior to an Entity. This is built for fixed framerate applications.
Otter.SineWaveComponent that controls a sine wave. Can be useful for special effects and such.
Otter.StateMachineComponent that runs a basic state machine.
Otter.StateMachine< TState >State machine that uses a specific type. This is really meant for using an enum as your list of states. If an enum is used, the state machine will automatically populate the states using methods in the parent Entity that match the name of the enum values.

Otter.EntityClass used for a game object. The bread and butter of your game. Entities are added to Scenes which are controlled by the Game.

Otter.ParticleEntity that is a quick way to make a particle. Has lots of parameters that can be set, so use this with that constructor where you do { } and put a bunch of properties inside. var p = new Particle(0, 0, "particle.png", 100, 100) { LifeSpan = 30, Angle = 10, FinalScaleX = 0, LockScaleRatio = true };
EventRouter.EventEvent data class, passed in to the subscriber whenever an event occours.
EventRouterClass for broadcasting messages to subscribers. Clients of the EventRouter register themselves by calling Subscribe and passing in the even they are interested in along with a delegate to be called back when the event is received. Events are published via the Publish method which allows arbitrary data to be passed along with the event to be interpreted by the event receiver. Events do not have to be pre-defined
